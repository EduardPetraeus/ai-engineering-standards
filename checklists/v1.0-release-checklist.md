# v1.0 Release Checklist

A comprehensive checklist for shipping v1.0 of any project. Every item
must be verified before tagging and publishing.

## Code Quality

- [ ] All tests pass (`python -m pytest tests/ -v`)
- [ ] Test coverage meets threshold (80%+ for v1.0)
- [ ] Ruff lint passes with zero errors (`ruff check .`)
- [ ] Ruff format passes (`ruff format --check .`)
- [ ] No `TODO` or `FIXME` comments without linked issues
- [ ] No `print()` statements in library code (use logging)
- [ ] No commented-out code blocks
- [ ] Type hints on all public function signatures
- [ ] All public functions have docstrings

## Testing

- [ ] Unit tests cover all core functionality
- [ ] Edge cases are tested (empty inputs, missing files, invalid data)
- [ ] CLI commands tested with Click's CliRunner
- [ ] Integration tests pass (if applicable)
- [ ] Tests run cleanly on a fresh environment

## CI/CD

- [ ] CI pipeline is green on the release branch
- [ ] Lint job passes
- [ ] Test job passes on all supported Python versions
- [ ] Self-validation job passes (if applicable)
- [ ] No flaky tests (re-run CI to confirm stability)

## Documentation

- [ ] README.md is complete and accurate
- [ ] Installation instructions work on a clean machine
- [ ] Usage examples are correct and copy-pasteable
- [ ] API documentation matches actual behavior
- [ ] CHANGELOG.md has a v1.0.0 entry with all changes
- [ ] All internal links resolve (no broken references)
- [ ] Architecture docs are current (if applicable)

## Standard Files

- [ ] README.md exists and is substantive
- [ ] LICENSE file exists with correct license text
- [ ] CLAUDE.md exists with current project context
- [ ] CONTRIBUTING.md explains how to contribute
- [ ] SECURITY.md has vulnerability reporting instructions
- [ ] CODE_OF_CONDUCT.md is present
- [ ] CHANGELOG.md tracks version history
- [ ] .gitignore covers build artifacts and secrets

## Configuration

- [ ] pyproject.toml has correct version number
- [ ] pyproject.toml has all required metadata (name, description, license)
- [ ] Dependencies are pinned to minimum versions
- [ ] Dev dependencies are in optional extras
- [ ] ruff.toml or [tool.ruff] is configured
- [ ] .pre-commit-config.yaml references pinned hook versions

## Security

- [ ] No secrets, API keys, or credentials in the codebase
- [ ] No PII in test data or documentation
- [ ] Dependencies have no known critical vulnerabilities
- [ ] Secret scanning is configured (detect-secrets or similar)
- [ ] SECURITY.md has responsible disclosure instructions

## Git and Release

- [ ] All work is on a feature branch (not main)
- [ ] Branch is up to date with main
- [ ] All commits follow conventional commit format
- [ ] PR description summarizes all changes
- [ ] Version number is bumped in all locations (__init__.py, pyproject.toml)
- [ ] Git tag matches version number (v1.0.0)
- [ ] No merge conflicts

## Post-Release

- [ ] Merge PR to main
- [ ] Create GitHub Release with changelog
- [ ] Verify package installs correctly from the release
- [ ] Update project status in INDEX.md / CONTEXT.md
- [ ] Close all related GitHub issues
- [ ] Notify stakeholders (if applicable)
